T1.number

给定一个 xxx ,与 aia_iai​,每一次可以对 xxx 操作为 x+aix+a_ix+ai​ 或 x−aix-a_ix−ai​,其中 每一轮 0≤x≤V,V≤10000 \le x \le V,V \le 10000≤x≤V,V≤1000

然后就可以设计一个简单模拟，将每一轮的可能都记录下来最多 100010001000 个，递推就可以取到最后的值

T2.transmit

就是给定一张图对于 disi,j≤didis_{i,j} \le d_idisi,j​≤di​ 的 jjj 点，可以花 tit_iti​ 的代价到达，求从 111 到达任一点的最小代价

然后可以用类似于 dijdijdij 的方法去做，然后对每一个 uuu 去更新时再一次 dijdijdij 找到符合条件的点

然后复杂度赛场没算，但想一想 222 种极端的情况一次更新很多，一次更新很少都可以比较快的处理，就直接交了，但赛场时思路比较乱，写了很久

T3.bread

就是对若干 aaa 有两种操作 a→⌊a2⌋a \to \lfloor \frac{a}{2} \rfloora→⌊2a​⌋ 或 a→⌈a2⌉a \to \lceil \frac{a}{2} \rceila→⌈2a​⌉

之前做 XiaoQuQuXiaoQuQuXiaoQuQu 的题有一道和这个类似，就是这个取值其实只有 log⁡V\log VlogV 种,做 jjj 次操作的数只可能是 ⌊ai2j⌋\lfloor \frac{a_i}{2^j} \rfloor⌊2jai​​⌋ 或者 ⌈ai2j⌉\lceil \frac{a_i}{2^j} \rceil⌈2jai​​⌉，然后答案必定是所有取值的交集中的一个，也就是最后的交集 SSS 必定满足 ∣S∣≤logV|S| \le log V∣S∣≤logV，然后比赛就结束了

但不过看完题解后发现就可以直接暴力做了好像是 Θ(nlog⁡2V)\Theta(n \log ^2 V)Θ(nlog2V)

但还有一种复杂度小一点的方法 Θ(nlog⁡V)\Theta(n\log V)Θ(nlogV)

就是按次数来一次一次处理，可以达到一致时就停止

T4.pancake

给定 n×mn \times mn×m 的点阵，问一条线最多可以切到几个点

暴力的话就是枚举线 Θ(n3m3)\Theta(n^3m^3)Θ(n3m3)

然后可以观察到一个性质当以 (x1,y1),(x2,y2)(x_1,y_1),(x_2,y_2)(x1​,y1​),(x2​,y2​) 为线的两端时，过点的数量就是 gcd⁡(∣x2−x1∣,∣y2−y1∣)+1,Θ(n2m2)\gcd(|x_2-x_1|,|y_2-y_1|)+1,\Theta(n^2m^2)gcd(∣x2​−x1​∣,∣y2​−y1​∣)+1,Θ(n2m2)

但是很多的线是重复算的，那么就可以枚举斜率，ab,gcd⁡(a,b)=1\frac{a}{b},\gcd(a,b)=1ba​,gcd(a,b)=1

若起点是 (x,y)(x,y)(x,y) 那终点必然是 (x+(k−1)a,y+(k−1)b),Θ(n2m2k2)(x+(k-1)a,y+(k-1)b),\Theta(\frac{n^2m^2}{k^2})(x+(k−1)a,y+(k−1)b),Θ(k2n2m2​)

然后考虑到 x+(k−1)a≤n<x+ka∨y+(k−1)b≤m<y+kb∨x≤a∨y≤bx+(k-1)a \le n < x+ka \lor y+(k-1)b \le m < y+kb \lor x \le a \lor y\le bx+(k−1)a≤n<x+ka∨y+(k−1)b≤m<y+kb∨x≤a∨y≤b 的条件就可以得到 n−ka<x≤n−(k−1)a∨m−kb<y≤m−(k−1)b∨x≤a∨y≤bn-ka < x \le n-(k-1)a \lor m-kb < y \le m-(k-1)b\lor x\le a \lor y\le bn−ka<x≤n−(k−1)a∨m−kb<y≤m−(k−1)b∨x≤a∨y≤b

然后计算就可以得到 Θ(nmlog⁡nk2)\Theta(\frac{nm\log n}{k^2})Θ(k2nmlogn​) 的复杂度
