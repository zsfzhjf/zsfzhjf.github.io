前言

这一次 ABCABCABC 相较于其他几次不是很好，只有 rnk2554rnk2554rnk2554，只有 +63+63+63，只在前 16min16min16min 切出 444 题，剩下的时间虽然再打但没 AAA 任何一道

A - 22222

给定字符串 SSS 求删去除 222 之外的字符的字符串 TTT 简单模拟，记录 222 的个数就行

B - cat

给出若干字符串 SSS，求按长度排序后合并的字符串

简单模拟

C - Debug

将 SSS 中所有的 WAWAWA 替换成 ACACAC

假设目前要改一个 WAWAWA

S1⋯SkWASk+3⋯Sn→S1⋯SkACSk+3⋯SnS_1 \cdots S_kWAS_{k+3}\cdots S_n \to S_1 \cdots S_kACS_{k+3}\cdots S_nS1​⋯Sk​WASk+3​⋯Sn​→S1​⋯Sk​ACSk+3​⋯Sn​

发现需要在做一次操作只可能发生在 SkAS_k ASk​A 上，每一次操作拨回指针重新判断就好

D - Colorful Bracket Sequence

问一个由 (,[,<(,[,<(,[,< 构成的序列是否满足类似于括号序列的条件

和括号序列一样用栈维护就好

E - Palindromic Shortest Path

问一个图，边权为 aaa 到 zzz 的字母，求任意两点的最短回文路径

可以依据回文性质分成奇回文串与偶回文串，分别考虑双向搜索， Θ(n4)\Theta(n^4)Θ(n4)

但很不幸，不仅 WAWAWA 还 TLETLETLE

其实可以考虑回文串的性质，不难联想到其是关于中心对称的，那么就可以用中心扩展法

由中心向两边扩展，BFSBFSBFS 更新答案就好

*这次知道了 AtcoderAtcoderAtcoder 用不了 LambdaLambdaLambda 表达式，会 CECECE

F - Alkane

给一个树，要求满足每一个点的度都是 111 或 444 且度为 444 的点必有一个的树的最大点数

没想，跳了

由于给的是一个无根树，那么一但确定了点集，那么树的形态也就确定了，考虑树上 dpdpdp

dpudp_udpu​ 表示以 uuu 为根的最大点数，考虑转移,简单讨论会发现只有度为 111 的烷其实只有 111 中，且点数为 222,而存在度为 444 的烷必定点数大于 444

发现在原先点集中加入一个 uuu 其实影响的只有与 uuu 有边的点

dpu=(∑v∈Sdpv)+1,∣S∣=3,dpi∈S>dpj∉Sdp_u = (\sum_{ v \in S} dp_v ) + 1 ,|S|=3,dp_{i \in S} > dp_{j \notin S} dpu​=(v∈S∑​dpv​)+1,∣S∣=3,dpi∈S​>dpj∈/S​

就是取 uuu 最大的 333 个孩子，在 faufa_ufau​ 还有一个度,然后是答案按照以 uuu 为根计算，可以分成 uuu 的度为 111 或 444 考虑

ans=max⁡{dpv+1,(∑v∈Sdpv)+1},∣S∣=4,dpi∈S>dpj∉Sans = \max \{ dp_v + 1 , (\sum_{ v \in S} dp_v ) + 1 \} ,|S|=4,dp_{i \in S} > dp_{j \notin S} ans=max{dpv​+1,(v∈S∑​dpv​)+1},∣S∣=4,dpi∈S​>dpj∈/S​

G - Dense Buildings

给定 H∗WH*WH∗W 的方阵 FFF

(i,j)(i,j)(i,j) 可达 (x,y)(x,y)(x,y) 当且仅当 h<Fx,yh < F_{x,y}h<Fx,y​ 其中 hhh 为目前高度，(i,j)(i,j)(i,j) 与 (x,y)(x,y)(x,y) 是四联通

每次给定起点与终点的三维坐标求最短距离

我当即拍案认为是 KruscalKruscalKruscal 重构树，但打出来后发现其不一定沿着走 FFF 最大的走

然后时间就快没了

赛后看完题解，原来自己是正解，看错题目了

不难发现从一点到另一点，有若干同意楼层上下移动的操作，而将其改为起点与终点上下移是不劣的因为初始高度越低可以直接走的可能性就越大

假设开始时下降到 MMM 那么答案就是 Y+Z−2MY+Z-2MY+Z−2M

由于不同楼层对答案没有影响，那么可以按照楼层高度高的来走

用 KruscalKruscalKruscal 重构树就可以找到最大的 MMM 简单判断一下就可以

结语

这一次时间分配不是很好，对于问题分析还没分析完手摸样例就开打，导致写完发现做法假的悲剧

下一次想好，手玩完样例再写代码

重点，看清题干
